1. System Overview & Goals

This manual documents everything required to design, develop, secure, deploy and operate the QGBC platform (Quantum Glass Business Cloud) and its AI assistant "Yusra". It is aimed at non-coders who will learn the system internals, and at engineers who will implement production-grade deliverables.

Primary objectives

  • Deliver a production-ready PWA and admin dashboard with real-time operations and a canonical accounting ledger (Postgres).
  • Provide an extensible AI layer for document processing, estimation and conversational assistance (Yusra).
  • Ensure reliability, observability, and guarded AI costs via canaries and quotas.

9. PostgreSQL DDL — Full Schema & Indexes

Run these DDL statements in your dev Postgres (managed Cloud SQL) as part of a migration. Use a migration tool (Flyway, Liquibase, or node-pg-migrate) to version these changes.

-- 001_create_accounts_and_core_tables.sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE companies (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  vat_no TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE projects (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  budget NUMERIC(18,2) DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE workers (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID NOT NULL REFERENCES companies(id),
  name TEXT NOT NULL,
  skillset JSONB,
  certifications JSONB,
  hourly_rate NUMERIC(12,2),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID NOT NULL REFERENCES companies(id),
  customer JSONB NOT NULL,
  total_amount NUMERIC(18,2) NOT NULL,
  currency TEXT NOT NULL DEFAULT 'SAR',
  status TEXT NOT NULL DEFAULT 'draft',
  idempotency_key TEXT UNIQUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE invoice_lines (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  invoice_id UUID NOT NULL REFERENCES invoices(id) ON DELETE CASCADE,
  description TEXT,
  qty NUMERIC(12,3) DEFAULT 1,
  unit_price NUMERIC(18,2) DEFAULT 0,
  tax_percent NUMERIC(5,2) DEFAULT 0
);

CREATE TABLE journal_entries (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  account_id UUID NOT NULL,
  debit NUMERIC(18,2) DEFAULT 0,
  credit NUMERIC(18,2) DEFAULT 0,
  txn_date DATE NOT NULL DEFAULT CURRENT_DATE,
  reference TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE INDEX idx_invoices_company_status ON invoices(company_id, status);
CREATE INDEX idx_projects_company ON projects(company_id);

Guidance: Use DB-level constraints and triggers to maintain balances or write a dedicated balancing service that posts counter-entries to the journal_entries table for every invoice payment.

10. Firestore Document Model & Security Rules (overview)

Use Firestore read-models to present denormalized, fast-loading documents for the UI. Keep sensitive financial fields authoritative in Postgres only.

// Suggested collections & doc structure
/companies/{companyId}
/projects/{projectId}
/projects/{projectId}/workers/{workerId}
/kpis/{companyId}
/invoices_view/{invoiceId}   // denormalized copy for UI & history
    {
      id, company_id, total_amount, status, created_at,
      lines: [{ description, qty, unit_price, tax_percent }],
      customer: {...}
    }

Security rules (example snippet)

// firestore.rules (high level)
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /companies/{companyId} {
      allow read: if request.auth != null && request.auth.token.company_id == companyId;
      allow write: if request.auth.token.role in ['admin','manager'] && request.auth.token.company_id == companyId;
    }

    match /invoices_view/{invoiceId} {
      allow read: if request.auth != null && request.auth.token.company_id == resource.data.company_id;
      allow write: if false; // only server-side mirroring updates this collection
    }
  }
}

11. Authentication & RBAC — Implementation Steps

  1. Use Firebase Auth for rapid multi-platform authentication (email, phone, Google). For enterprise SSO later, integrate SAML/OIDC provider.
  2. Assign roles as custom claims: role (admin, manager, accountant, worker) and company_id. Custom claims are used in server-side checks and Firestore rules.
  3. Server functions validate the token on every request and re-check claims; do not trust client-side role information.
  4. Maintain an audit trail: every sensitive action (role change, financial export) must be recorded in an audit table in Postgres and optionally mirrored to a secure audit log storage.

14. Termux Bootstrap Script (Downloadable)

Save the script below as bootstrap_qgbc.sh on your Termux device, mark executable (chmod +x), edit variables, and run. It installs toolchain, clones scaffold, and starts the Firebase emulator (if configured).

#!/data/data/com.termux/files/usr/bin/bash
set -euo pipefail
# Config — edit before running
PROJECT_ID="qgbc-dev"
GIT_REPO="git@github.com:yourname/qgbc.git"
REPO_DIR="$HOME/qgbc"
NPM_GLOBAL_PREFIX="$HOME/.npm-global"
NODE_VERSION="20"
# Update & packages
pkg update -y && pkg upgrade -y
pkg install -y git nodejs python clang make openssh wget
# npm global prefix
mkdir -p "$NPM_GLOBAL_PREFIX"
npm config set prefix "$NPM_GLOBAL_PREFIX"
export PATH="$NPM_GLOBAL_PREFIX/bin:$PATH"
echo 'export PATH="'"$NPM_GLOBAL_PREFIX"'/bin:$PATH"' >> ~/.profile
# Install utilities
npm install -g pnpm firebase-tools@11.18.0
# Clone or update repo
if [ -d "$REPO_DIR" ]; then
  cd "$REPO_DIR" && git pull
else
  git clone "$GIT_REPO" "$REPO_DIR"
  cd "$REPO_DIR"
fi
# Install dependencies
pnpm install
# Firebase login: do on desktop and set FIREBASE_TOKEN env if headless
echo "If browser login required run: firebase login"
# Start emulator
firebase emulators:start --only auth,firestore,functions,hosting --project "$PROJECT_ID"

21. Example: End-to-end flow — Create Invoice (detailed)

  1. Client UX creates invoice draft and requests creation with idempotency_key.
  2. API validates and forwards to an invoices.create Cloud Function.
  3. Function starts DB transaction in Postgres, inserts invoice, computes totals, inserts invoice_lines, commits.
  4. Function publishes invoice.created event to Pub/Sub.
  5. Mirror worker subscribes to the topic, denormalizes the invoice to Firestore for UI and search.
  6. Notify accounts via email/SMS and add entry to audit table.

  const token = await getAuthToken();

export async function postWithRetry(url, data, retries = 2){
  for(let i=0;i<=retries;i++){
    try { return await api.post(url, data); }
    catch(err){ if(i===retries) throw err; await new Promise(r=>setTimeout(r, 500*(i+1))); }
  }
}

34. Example Cloud Function (JS) — invoice.create

// functions/src/handlers/invoices.js (simplified)
import { pool } from '../lib/db';
import { pubsubClient } from '../lib/pubsub';

export async function createInvoice(req, res){
  const { idempotency_key, company_id, customer, lines } = req.body;
  // validate...
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    // check idempotency
    const r = await client.query('SELECT id FROM invoices WHERE idempotency_key=$1', [idempotency_key]);
    if(r.rowCount) { await client.query('COMMIT'); return res.json({ id: r.rows[0].id }); }
    const insert = await client.query(
      'INSERT INTO invoices(company_id, customer, total_amount, idempotency_key) VALUES($1,$2,$3,$4) RETURNING id',
      [company_id, customer, /* compute total */, idempotency_key]
    );
    const invoiceId = insert.rows[0].id;
    // insert lines...
    await client.query('COMMIT');
    await pubsubClient.publish('invoice.created', { invoiceId, company_id });
    res.json({ id: invoiceId });
  } catch(e){
    await client.query('ROLLBACK'); throw e;
  } finally { client.release(); }
}

১. সিস্টেমের সংক্ষিপ্ত বিবরণ ও লক্ষ্যসমূহ

এই ম্যানুয়ালটি QGBC প্ল্যাটফর্ম (কোয়ান্টাম গ্লাস বিজনেস ক্লাউড) এবং এর এআই সহকারী "ইউসরা"-কে ডিজাইন, ডেভেলপ, সুরক্ষিত, ডিপ্লয় এবং পরিচালনা করার জন্য প্রয়োজনীয় সবকিছু নথিভুক্ত করে। এটি নন-কোডারদের জন্য যারা সিস্টেমের অভ্যন্তরীণ বিষয়গুলো জানতে চান এবং ইঞ্জিনিয়ারদের জন্য যারা প্রোডাকশন-গ্রেড সরবরাহযোগ্যগুলি বাস্তবায়ন করবেন।

প্রাথমিক লক্ষ্য

  • রিয়েল-টাইম অপারেশন এবং একটি ক্যানোনিকাল অ্যাকাউন্টিং লেজার (পোস্টগ্রেস) সহ একটি প্রোডাকশন-রেডি PWA এবং অ্যাডমিন ড্যাশবোর্ড সরবরাহ করা।
  • ডকুমেন্ট প্রক্রিয়াকরণ, অনুমান (estimation) এবং কথোপকথনমূলক সহায়তার জন্য একটি এক্সটেনসিবল এআই স্তর (ইউসরা) সরবরাহ করা।
  • ক্যানারি এবং কোটার মাধ্যমে নির্ভরযোগ্যতা, পর্যবেক্ষণযোগ্যতা (observability), এবং সুরক্ষিত এআই খরচ নিশ্চিত করা।

২. উচ্চ-স্তরের আর্কিটেকচার ডায়াগ্রাম

৫. ডেভেলপমেন্টের ক্রম — ৪০-ধাপের সময়রেখা (সংক্ষিপ্ত)

শূন্য থেকে প্রোডাকশনে যাওয়ার জন্য ধাপগুলির একটি উচ্চ-রেজোলিউশন ক্রম নিচে দেওয়া হলো। প্রতিটি আইটেমকে আপনার ইস্যু ট্র্যাকারে একটি কাজ হিসাবে ট্র্যাক করা উচিত এবং এর গ্রহণযোগ্যতার মানদণ্ড (acceptance criteria) থাকা উচিত।

  1. ডোমেন মডেল ও ম্যাপিং সংজ্ঞায়িত করুন: কোন সত্তাগুলি পোস্টগ্রেস বনাম ফায়ারস্টোরে থাকবে তা নির্ধারণ করুন।
  2. মনোরেপো শুরু করুন, ওয়ার্কস্পেস তৈরি করুন, প্রাথমিক লকফাইলগুলি কমিট করুন।
  3. ক্লাউড প্রোজেক্টগুলি সরবরাহ করুন: Terraform কঙ্কাল ব্যবহার করে dev, stage, prod।
  4. CI কঙ্কাল সেট আপ করুন: lint, typecheck, unit tests, build।
  5. অথ কঙ্কাল (Firebase Auth) এবং RBAC ডিজাইন ডকুমেন্ট বাস্তবায়ন করুন।
  6. ফ্রন্টএন্ড শেল এবং ডিজাইন সিস্টেম তৈরি করুন (Tailwind টোকেন + Storybook)।
  7. পোস্টগ্রেস স্কিমা DDL তৈরি করুন এবং ডেভ ডিবিতে মাইগ্রেশন চালান।
  8. ফায়ারস্টোর ডকুমেন্ট মডেল তৈরি করুন এবং নিরাপত্তা নিয়মাবলী লিখুন; এমুলেটর দিয়ে পরীক্ষা করুন।
  9. ইনভয়েস এপিআই বাস্তবায়ন করুন (আইডেমপোটেন্ট, পোস্টগ্রেসে লেখে)।
  10. মিরর জব বাস্তবায়ন করুন: invoice.created → Firestore invoice_view।
  11. WPS এক্সপোর্ট মডিউল সহ পেরোল ইন্টিগ্রেশন বাস্তবায়ন করুন।
  12. কর্মী তালিকা (worker roster) এবং সার্টিফিকেট আপলোড ফ্লো বাস্তবায়ন করুন।
  13. সিইও ড্যাশবোর্ড এবং কেপিআই পুনঃগণনা জব তৈরি করুন (নির্ধারিত)।
  14. এআই ইনজেশন পাইপলাইন বাস্তবায়ন করুন (OCR -> এম্বেডিংস -> ভেক্টর স্টোর)।
  15. রেট-লিমিট এবং ক্যাচিং সহ এআই ক্যোয়ারী এন্ডপয়েন্ট ইন্টিগ্রেট করুন।
  16. গুরুত্বপূর্ণ ফ্লোগুলির জন্য এমুলেটরের বিরুদ্ধে ইন্টিগ্রেশন পরীক্ষা চালান।
  17. প্রিভিউ চ্যানেল ডিপ্লয় করুন এবং ম্যানুয়ালি স্মোক পরীক্ষা চালান।

  19. লোড পরীক্ষা চালান এবং ইনডেক্স এবং ফাংশন স্কেলিং টিউন করুন।
  20. নিরাপত্তা নিরীক্ষা এবং পেন-টেস্ট সংশোধন সম্পাদন করুন।

-- 001_create_accounts_and_core_tables.sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE companies (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  vat_no TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE projects (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  budget NUMERIC(18,2) DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE workers (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID NOT NULL REFERENCES companies(id),
  name TEXT NOT NULL,
  skillset JSONB,
  certifications JSONB,
  hourly_rate NUMERIC(12,2),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  company_id UUID NOT NULL REFERENCES companies(id),
  customer JSONB NOT NULL,
  total_amount NUMERIC(18,2) NOT NULL,
  currency TEXT NOT NULL DEFAULT 'SAR',
  status TEXT NOT NULL DEFAULT 'draft',
  idempotency_key TEXT UNIQUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE TABLE invoice_lines (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  invoice_id UUID NOT NULL REFERENCES invoices(id) ON DELETE CASCADE,
  description TEXT,
  qty NUMERIC(12,3) DEFAULT 1,
  unit_price NUMERIC(18,2) DEFAULT 0,
  tax_percent NUMERIC(5,2) DEFAULT 0
);

CREATE TABLE journal_entries (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  account_id UUID NOT NULL,
  debit NUMERIC(18,2) DEFAULT 0,
  credit NUMERIC(18,2) DEFAULT 0,
  txn_date DATE NOT NULL DEFAULT CURRENT_DATE,
  reference TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
);

CREATE INDEX idx_invoices_company_status ON invoices(company_id, status);
CREATE INDEX idx_projects_company ON projects(company_id);

১১. অথেন্টিকেশন ও আরবিএসি — বাস্তবায়নের ধাপসমূহ

  1. দ্রুত মাল্টি-প্ল্যাটফর্ম অথেন্টিকেশন (ইমেল, ফোন, গুগল) এর জন্য ফায়ারবেস অথ ব্যবহার করুন। পরবর্তীকালে এন্টারপ্রাইজ এসএসও-এর জন্য, SAML/OIDC প্রদানকারীকে ইন্টিগ্রেট করুন।
  2. কাস্টম ক্লেইম হিসাবে ভূমিকা নির্ধারণ করুন: role (admin, manager, accountant, worker) এবং company_id। কাস্টম ক্লেইমগুলি সার্ভার-সাইড চেক এবং ফায়ারস্টোর নিয়মাবলীতে ব্যবহৃত হয়।
  3. সার্ভার ফাংশনগুলি প্রতিটি অনুরোধে টোকেন যাচাই করে এবং ক্লেইমগুলি পুনরায় পরীক্ষা করে; ক্লায়েন্ট-সাইড ভূমিকার তথ্যের উপর আস্থা রাখবেন না।
  4. একটি অডিট ট্রেইল বজায় রাখুন: প্রতিটি সংবেদনশীল ক্রিয়া (ভূমিকা পরিবর্তন, আর্থিক এক্সপোর্ট) পোস্টগ্রেসে একটি অডিট টেবিলে রেকর্ড করা আবশ্যক এবং ঐচ্ছিকভাবে একটি সুরক্ষিত অডিট লগ স্টোরেজে মিরর করা যেতে পারে।

#!/data/data/com.termux/files/usr/bin/bash
set -euo pipefail
# Config — edit before running
PROJECT_ID="qgbc-dev"
GIT_REPO="git@github.com:yourname/qgbc.git"
REPO_DIR="$HOME/qgbc"
NPM_GLOBAL_PREFIX="$HOME/.npm-global"

# Update & packages
pkg update -y && pkg upgrade -y
pkg install -y git nodejs python clang make openssh wget
# npm global prefix
mkdir -p "$NPM_GLOBAL_PREFIX"

export PATH="$NPM_GLOBAL_PREFIX/bin:$PATH"
echo 'export PATH="'"$NPM_GLOBAL_PREFIX"'/bin:$PATH"' >> ~/.profile
# Install utilities
npm install -g pnpm firebase-tools@11.18.0
# Clone or update repo
if [ -d "$REPO_DIR" ]; then
  cd "$REPO_DIR" && git pull
else

fi

pnpm install

firebase emulators:start --only auth,firestore,functions,hosting --project "$PROJECT_ID"

-- Monthly revenue by company (প্রতি কোম্পানি মাসিক রাজস্ব)
SELECT company_id, date_trunc('month', created_at) AS month, SUM(total_amount) AS revenue
FROM invoices
WHERE status = 'paid'
GROUP BY company_id, month
ORDER BY month DESC;

-- Worker profitability: revenue attributed / total cost (কর্মী লাভজনকতা: আরোপিত রাজস্ব / মোট খরচ)

import axios from 'axios';
import { getAuthToken } from './auth';

const api = axios.create({ baseURL: process.env.API_URL, timeout:15000 });

api.interceptors.request.use(async config => {
  const token = await getAuthToken();
  if(token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

export async function postWithRetry(url, data, retries = 2){
  for(let i=0;i<=retries;i++){
    try { return await api.post(url, data); }
    catch(err){ if(i===retries) throw err; await new Promise(r=>setTimeout(r, 500*(i+1))); }
  }
}

// functions/src/handlers/invoices.js (simplified)
import { pool } from '../lib/db';
import { pubsubClient } from '../lib/pubsub';

export async function createInvoice(req, res){
  const { idempotency_key, company_id, customer, lines } = req.body;
  // validate...
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    // check idempotency
    const r = await client.query('SELECT id FROM invoices WHERE idempotency_key=$1', [idempotency_key]);
    if(r.rowCount) { await client.query('COMMIT'); return res.json({ id: r.rows[0].id }); }
    const insert = await client.query(
      'INSERT INTO invoices(company_id, customer, total_amount, idempotency_key) VALUES($1,$2,$3,$4) RETURNING id',
      [company_id, customer, /* compute total */, idempotency_key]
    );
    const invoiceId = insert.rows[0].id;
    // insert lines...
    await client.query('COMMIT');
    await pubsubClient.publish('invoice.created', { invoiceId, company_id });
    res.json({ id: invoiceId });
  } catch(e){
    await client.query('ROLLBACK'); throw e;
  } finally { client.release(); }
}
